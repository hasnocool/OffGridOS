# 4. Development Roadmap

## Phase 0 — Foundation

Define naming, repository standards, supported hardware, threat model, data ownership rules, and a reproducible development environment.

## Phase 1 — OMS Alpha

Deliver an installable Media Server image or installer for a small set of hardware. Include local dashboard, media service, file storage, knowledge vault, offline maps, backup, health monitoring, and documentation.

## Phase 2 — Household platform

Add local AI, device discovery, role-based access, energy telemetry, notifications, safer update workflows, and a polished admin experience.

## Phase 3 — Community Atlas

Add trusted-location records, service listings, access rules, reputation signals, moderation, map layers, and privacy-preserving synchronization.

## Phase 4 — Resilient networks

Integrate mesh networking, point-to-point links, radio gateways, store-and-forward messaging, and community deployment tooling.

## Phase 5 — Marketplace and incentives

Introduce service exchange, verified participation credits, optional token settlement, governance, and partner APIs only after identity, abuse prevention, and accounting are proven.

## Release gates

Every phase requires functional tests, backup/restore tests, security review, power and storage measurements, documentation review, and a rollback path.
